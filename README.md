# pro-flowers

Frontend заготовка для интернет-магазина цветов с дальнейшей интеграцией на WordPress (CMS).

---

## 🚀 Установка и запуск проекта

Установка зависимостей:
npm install

Запуск режима разработки:
npm run dev

---

## 📦 Сборка проекта

Сборка продакшн версии:
npm run build

---

## 📁 Структура страниц (HTML)

### Основные страницы
front-page.html — главная страница  
catalog.html — каталог товаров  
product.html — карточка товара (шаблон под все товары)  
services.html — услуги (с попапом формы заказа услуги)

---

### Информационные страницы
delivery.html — условия доставки  
payment.html — условия оплаты  
about.html — о компании  
contacts.html — контакты  
guarantee.html — гарантия на товар  
offer.html — публичная оферта (договор)  
privacy.html — политика конфиденциальности

---

### Блог
blog.html — список новостей  
single.html — страница одной новости

---

### Магазин
cart.html — корзина  
thank-you.html — страница “спасибо за заказ”

---

### Личный кабинет
login.html — авторизация  
register.html — регистрация  
forgot-password.html — восстановление пароля  
account.html — личный кабинет (внутри: личные данные + заказы)

---

### Служебные страницы
404.html — страница ошибки

---

## 🔁 Важная архитектурная логика (WordPress)

При переносе на WordPress структура упрощается:

- page.php → единый шаблон для всех страниц:
    - доставка
    - оплата
    - о компании
    - контакты
    - гарантия
    - оферта
    - политика

- single.php → карточка товара / новость
- archive.php → каталог / блог
- front-page.php → главная
- woocommerce (или кастом) → корзина / оформление заказа

---

## 🧠 Логика проекта

- product.html = универсальный шаблон товара
- services.html = услуги с модальным окном заказа
- account.html = личный кабинет с вкладками:
    - личные данные (редактирование)
    - заказы (история + статус)

---

## ⚙️ Важно

Этот проект — HTML-сборка перед натяжкой на WordPress.  
Все страницы делаются максимально модульно, чтобы:

- легко перенести в PHP-шаблоны
- минимально переписывать верстку
- переиспользовать блоки

---

## 📌 Итог

После верстки:
- HTML → превращается в WordPress шаблоны
- page.html → page.php
- product.html → single-product.php
- blog.html → archive.php + single.php
